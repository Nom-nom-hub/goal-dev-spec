Local Development Guide

Contributing to and developing with Goal Kit

Setting Up Local Development Environment

Goal Kit is an open-source project that welcomes contributions from the community. This guide will help you set up a local development environment to contribute or modify the framework for your own needs.

Cloning the Repository

Start by cloning the Goal Kit repository from GitHub:

$ git clone https://github.com/Nom-nom-hub/goal-kit.git
$ cd goal-kit

Dependency Management

Goal Kit uses uv for dependency management. After cloning, set up your development environment:

$ uv sync

This command creates a virtual environment and installs all dependencies including development tools.

Running in Development Mode

You can run Goal Kit commands directly in development mode:

$ uv run goal init <PROJECT_NAME>

Or activate the virtual environment to use Goal Kit commands directly:

$ source .venv/bin/activate  # On Windows: .venv\Scripts\activate
$ goal init <PROJECT_NAME>

Running Tests

Before submitting changes, ensure all tests pass:

$ uv run pytest

For more detailed test output:

$ uv run pytest -v

Code Formatting and Linting

Goal Kit uses Ruff for formatting and linting. Format your code before submitting:

$ uv run ruff format .
$ uv run ruff check .

Building Documentation

When making changes to documentation, you can serve it locally:

$ uv run mkdocs serve

Submitting Changes

When you're ready to contribute changes:

  1. Create a new branch: git checkout -b feature/your-feature-name
  2. Make your changes
  3. Ensure all tests pass
  4. Format your code
  5. Commit your changes with clear messages
  6. Push your branch and create a pull request

Development Workflow

When developing with Goal Kit for your own projects, you can create a local copy of the framework with your customizations:

  1. Fork the repository on GitHub
  2. Clone your fork locally
  3. Make your modifications
  4. Test your changes with the example commands
  5. Publish your customized version if needed

Advanced Development

For more advanced development, you can also:

  • Extend the CLI with new commands
  • Add new templates for different project types
  • Modify the goal evaluation algorithms
  • Create new output formats

Next Steps

Now that you have set up your local development environment, check out our Home Page to learn more about the framework or our Quick Start Guide to begin using the framework.
