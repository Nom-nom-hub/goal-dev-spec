Tutorials & Guides

Step-by-step tutorials to help you master Goal-Dev-Spec and goal-driven development.

Quick Start Tutorial

Get up and running with Goal-Dev-Spec in just a few minutes. 

# 1. Install Goal-Dev-Spec
pip install -e .

# 2. Initialize a new project
goal init my-awesome-project

# 3. Navigate to your project
cd my-awesome-project

# 4. Create your first goal
goal create "Implement user authentication with login and registration"

# 5. List your goals to get the ID
goal list

# 6. Create an implementation plan (replace GOAL_ID with the actual ID)
goal plan GOAL_ID

# 7. Generate task breakdown
goal tasks PLAN_ID

# 8. Track progress
goal track

What You've Accomplished

In just a few commands, you've:

  • Created a structured project with proper organization
  • Defined a clear goal with measurable objectives
  • Generated a detailed implementation plan
  • Broken down the plan into actionable tasks
  • Set up progress tracking for your project

Learn to structure web application projects with Goal-Dev-Spec.

Steps:

  1. Define goals for core features (authentication, dashboard, API)
  2. Create specifications with user stories and requirements
  3. Plan frontend and backend components separately
  4. Generate tasks for each development phase

Best Practices & Tips

Tips and strategies from experienced users of Goal-Dev-Spec.

Goal Definition

Write clear, measurable goals that align with business objectives.

  • Be specific and avoid vague language
  • Define measurable success criteria
  • Identify stakeholders and dependencies
  • Set realistic timelines

Task Management

Effectively break down plans into manageable tasks.

  • Keep tasks small and achievable
  • Define clear acceptance criteria
  • Set appropriate priorities
  • Track dependencies carefully

Team Collaboration

Work effectively with teams using Goal-Dev-Spec.

  • Use version control for goal files
  • Define clear roles and responsibilities
  • Establish regular review cycles
  • Share knowledge through specifications
